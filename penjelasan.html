GameZone — Sistem Rental Warnet & PS

Penjelasan Project untuk Presentasi

GameZone adalah sistem pemesanan (booking) perangkat game seperti PlayStation dan PC Gaming di sebuah rental/warnet.

Apa yang bisa dilakukan?

Pelanggan:

  • Lihat daftar perangkat yang tersedia
  • Pesan perangkat untuk waktu tertentu
  • Bayar langsung & dapat tiket (QR)
  • Ikut turnamen & ambil promo
  • Cek riwayat & dompet poin (wallet)

Admin & Staff: kelola perangkat, pengguna, pembayaran, laporan, & pengaturan.

Dibuat dengan PHP Native & MySQL — tanpa framework berat, jadi ringan & mudah dipahami.

2. Struktur Project = Seperti Kantor

Memudahkan memahami fungsi tiap folder

Bayangkan project ini seperti sebuah kantor:

  • Resepsionis → halaman depan (index.php, login.php)
  • Ruang kerja tiap divisi → folder modules/
  • Brankas data → database MySQL (lewat config/database.php)
  • Buku panduan umum → folder inc/
  • Pelayan data otomatis → folder api/

4. Admin, API & Config

"Ruang Karyawan, Pelayan Data, & Kunci Brankas"
admin/

index.php — Login khusus admin & staff. Customer tidak boleh masuk lewat sini.

api/

Endpoint yang mengembalikan data JSON (bukan halaman), dipakai tombol dashboard secara otomatis tanpa reload:

8. Cara Kerja Sistem (Alur)

Dari buka website sampai dapat QR ticket
  1. Pelanggan buka index.php → lihat device
  2. Login → masuk dashboard.php
  3. Pilih Booking: device, waktu, durasi, metode bayar
  4. Sistem validasi → cek device tersedia → hitung total & diskon
  5. Klik bayar → dalam 1 transaksi DB: booking tersimpan, device "dibooking", invoice & payment otomatis dibuat
  6. Pelanggan dapat QR ticket untuk check-in
  7. Staff ubah status via API → device kembali "tersedia" & poin bertambah

9. Teknologi yang Dipakai

"Peralatan yang digunakan"
Frontend: HTML, CSS, JS Bootstrap 5 Tailwind Backend: PHP Native 8+ Database: MySQL / MariaDB PDO (aman)

Keamanan: password di-hash, token CSRF, session HttpOnly + SameSite.

10. Poin Keamanan

Hal penting untuk disebut ke dosen
  • Password aman: di-hash, tidak disimpan mentah (password_hash)
  • Anti manipulasi form: tiap aksi penting pakai CSRF token
  • Pembagian hak akses: customer tak bisa buka menu karyawan (requireAdmin dll)
  • Transaksi database: booking pakai BEGIN … COMMIT agar data tak setengah jadi
  • Kunci ganda turnamen: FOR UPDATE agar kuota tak bentrok saat 2 orang daftar bersamaan
